§ 00 — 先把全局放對

OCI 有三個規範，本篇聚焦Runtime Spec

OCI（Open Container Initiative）把「容器」拆成三段標準化：Distribution（怎麼傳）、Image（怎麼存/長什麼樣）、Runtime（怎麼跑）。

這篇教學只做一件事：把 OCI Runtime Spec 從抽象文字變成你能操作的流程，最後用 runc 從 0 跑起一個最小容器。

理論 → 實作 以 Linux 為主 runc hands-on

§ 00.75 — Runtime Spec 在規範什麼？

三個核心：Bundle、State、Lifecycle

你只要記住：runtime 吃的是 bundle，吐出的是可觀測的 state，並且遵守一組可操作的 lifecycle。點按鈕看對應即可。

// 先有 bundle，才談 runtime
// 上層工具（containerd/CRI-O）做的：
① unpack image → 組成 rootfs
② 生成 config.json → 組成 bundle

// runtime（runc/crun）做的：
③ 依 config 建立 namespace / cgroups / mounts / seccomp
④ 啟動 process 並維護狀態 + hooks
讀到這裡你只要抓住：OCI Runtime 的輸入是 bundle，輸出是「一個被隔離與限制的程序」。

用按鈕切換：每個概念都會給你「它在解決什麼」以及「在 runc / config.json 對應到哪裡」。

§ 05 — cgroups 資源限制

資源的水桶模型

Control Groups（cgroups）限制容器能使用的 CPU、Memory、I/O 等資源。想像每種資源是一個有容量上限的水桶，程序只能用桶內的量。

超過警戒線（80%）系統發出警告，超過上限容器行程會被強制終止（OOM Kill）。

拖動滑桿來調整資源使用量，觀察水桶反應。

§ 06 — Seccomp & Capabilities

系統呼叫過濾牆

Seccomp（Secure Computing Mode）在容器的 App 與 Linux Kernel 之間建立一道過濾牆，只允許白名單內的系統呼叫通過，其餘一律拒絕。

Capabilities 則更細緻地控制 root 的特定特權，例如允許 CAP_NET_BIND_SERVICE 但拒絕 CAP_SYS_ADMIN。

§ 07 — 容器生命週期

從 creating 到 stopped

這裡是 Runtime Spec 最「可操作」的一塊：它把容器抽象成一個狀態機（creating/created/running/stopped），並規範一組標準操作。你在這裡按的每個按鈕，都能在 §10 直接用對應的 runc 指令做出一樣的效果。

🔨
creating
📦
created
▶️
running
⏹️
stopped
// runc 對照（同一個 lifecycle）
$ sudo runc create myct   # initial → created（建立隔離環境，程序未啟動）
$ sudo runc start myct   # created → running
$ sudo runc kill myct TERM   # running → stopped
$ sudo runc delete myct   # stopped → (removed)
初始狀態 — 點擊 create 開始建立容器

  "ociVersion": "1.1.0",

  "status": "–",
  "pid": 0,

}

§ 10 — runc 實戰（Ubuntu 完整流程）

與教材同級的 bundle + runc run 實作

下面兩塊並行：可捲動的完整指令（在真機 Ubuntu 上照抄即可）與 左側步驟 + 右側終端模擬（對照意義）。流程對齊常見教學：準備 rootfs → runc spec 編修 → sudo runc run bb8 → 在 shell 裡驗證隔離與 /proc 行為。

// 完整手動流程（Ubuntu 22.04 / 24.04，於宿主機終端機執行）
0 — 安裝依賴

需要 root。若用 Docker 匯出 rootfs，請裝 Docker；若不用 Docker，可改走「方案 B」Alpine tarball。

sudo apt update
sudo apt install -y runc curl jq nano ca-certificates iproute2
# 選用：從 Docker 映像匯出 rootfs（與多數教學相同）
sudo apt install -y docker.io
sudo systemctl enable --now docker
1 — 建立 OCI bundle 目錄
mkdir -p ~/oci-bb8/rootfs
cd ~/oci-bb8
2 — 填入 rootfs（擇一）

方案 A（BusyBox，需 Docker） — 與截圖中 /bin → busybox 行為一致。

sudo docker create --name bb-export busybox:latest sh
sudo docker export bb-export | sudo tar -C rootfs -xf -
sudo docker rm bb-export
test -x rootfs/bin/sh && echo "OK: rootfs/bin/sh"

方案 B（Alpine minirootfs，免 Docker） — 類似你另一張圖：下載官方 minirootfs 再解到 rootfs/。請到 Alpine 發行目錄 換成目前穩定版連結。

curl -fsSL -o alpine.tar.gz "https://dl-cdn.alpinelinux.org/alpine/v3.19/releases/x86_64/alpine-minirootfs-3.19.3-x86_64.tar.gz"
sudo tar xzf alpine.tar.gz -C rootfs
rm -f alpine.tar.gz
test -x rootfs/bin/sh && echo "OK: rootfs/bin/sh"

方案 C（极小 Ubuntu rootfs，較大下載）

sudo apt install -y debootstrap
sudo debootstrap jammy rootfs http://archive.ubuntu.com/ubuntu/
3 — 將 rootfs 擁有者改為 root（runc 常要求·與教學說明一致）

若你是用一般使用者下載解壓，目錄可能是 ubuntu:ubuntu；建議整個 bundle 交給 root。

sudo chown -R root:root ~/oci-bb8
4 — 產生並編輯 OCI config.json

對齊教學範例：hostname dagger、互動 shell、args: ["sh"]、root 指向 rootfs。runc spec 會帶入預設 namespaces 與 /proc 掛載。

cd ~/oci-bb8
sudo runc spec

用 nano 手改，或一次性用 jq（擇一）：

sudo nano config.json
sudo cp config.json config.json.bak
sudo jq '.hostname = "dagger"
  | .process.terminal = true
  | .process.args = ["sh"]
  | .process.user = {"uid": 0, "gid": 0}
  | .root = (.root // {}) | .root.path = "rootfs" | .root.readonly = false' config.json.bak > config.json
5 — 以互動方式啟動容器（對齊 runc run bb8）
cd ~/oci-bb8
sudo runc run bb8

應進入容器內 shell（提示字可能類似 / #）。離開表示結束該次容器行程：exit。

6 — 在容器內驗證隔離（與教材截圖對應）
hostname
ip a s
ps aux
ls -al /bin | head -n 8

預期：hostname 為 dagger；網路命名空間隔離時通常只見 lo；ps 中 sh 為 PID 1；用 BusyBox/Alpine 時 /bin 底下常見指向 busybox 的 symlink。

7 — rootfs 會被寫入；虛擬檔案受核心保護

在容器內仍是 root，變更會寫回宿主機上的同一份 rootfs/，因此不適合再給「另一個乾淨容器」共用同一路徑而不重建。/proc 下為核心介面，刪除會失敗。

whoami
mkdir /zzz
rm /bin/zcat
cat /proc/meminfo | head -n 3
rm /proc/meminfo

預期：rm /proc/meminfo 出現 Permission denied（或類似錯誤）。離開：exit。

8 — 進階：拆步生命週期（可選，對照本頁 §07）
cd ~/oci-bb8
sudo runc create myct
sudo runc state myct
sudo runc start myct
sudo runc kill myct KILL
sudo runc delete myct
